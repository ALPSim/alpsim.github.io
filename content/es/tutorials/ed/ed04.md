En este tutorial, examinaremos cadenas de espín críticas y estableceremos una conexión con su descripción en términos de teoría de campos conforme.

## Cadena de Ising

El primer modelo que consideraremos es la cadena de Ising crítica, dada por el hamiltoniano

$$
H=J_{z} \sum_{\langle i,j \rangle} S^i_z S^j_z + \Gamma \sum_i S^i_x
$$

Aquí, la primera suma recorre pares de vecinos más cercanos. $\Gamma$ se denomina campo transverso; el sistema se vuelve crítico para $\Gamma/J=\frac{1}{2}$. Para $\Gamma=0$, el estado fundamental es antiferromagnético para $J\gt 0$ y ferromagnético para $J \lt 0$. El sistema es exactamente soluble ([P. Pfeuty, Annals of Physics: 57, 79-90 (1970)](https://www.sciencedirect.com/science/article/abs/pii/0003491670902708?via%3Dihub)).

En la ecuación anterior, $\Delta$ se refiere a la dimensión de escala de ese campo. Los campos de escala aparecen en grupos: el más bajo, denominado campo primario, viene acompañado de un número infinito de descendientes con dimensión de escala $\Delta + m$, $m \in \lbrace 1, 2, 3, ... \rbrace$.

En la solución exacta del modelo de Ising (Ec. (3.7) en [el artículo de P. Pfeuty](https://www.sciencedirect.com/science/article/abs/pii/0003491670902708?via%3Dihub)), se encuentra que las correlaciones de largo alcance decaen como:
$$
\langle S^i_z S^{i+n}_z \rangle \sim n^{-2\times 1/8}
$$
$$
\langle S^i_y S^{i+n}_y \rangle \sim n^{-2\times(1+1/8)}
$$
$$
\langle S^i_x S^{i+n}_x \rangle \sim n^{-2\times 1}
$$
Además, esperamos que la dimensión de escala del operador identidad sea 0.

Por lo tanto, esperamos que aparezcan dimensiones de escala de 0, 1/8, 1, 1+1/8 en la TCC del modelo de Ising. Para verlo, reescalaremos todas las energías del espectro según $E \rightarrow \frac{E-E_0}{(E_1-E_0)8}$. Esto forzará a que los dos estados más bajos aparezcan donde esperamos las dimensiones de escala; luego podemos comprobar si el resto del espectro es consistente con esto.

### Versión en Python

El archivo de parámetros para la simulación de Ising se puede encontrar <a href="../codes/ed-04-criticality/parm_ising" download>aquí</a>. La simulación se ejecuta con el script de Python.

Primero importaremos algunos módulos:

```
import pyalps
import pyalps.plot
import numpy as np
import matplotlib.pyplot as plt
import copy
import math
```
    
Luego, configuremos los parámetros para dos tamaños de sistema. Tenga cuidado de usar el campo transverso $\Gamma$, no el campo longitudinal $h$.

```
parms = []
for L in [10,12]:
    parms.append({
        'LATTICE'    : "chain lattice",
        'MODEL'      : "spin",
        'local_S'    : 0.5,
        'Jxy'        : 0,
        'Jz'         : -1,
        'Gamma'      : 0.5,
        'NUMBER_EIGENVALUES' : 5,
        'L'          : L
    })
```

Como puede ver, simularemos dos tamaños de sistema. Ahora configuremos los archivos de entrada y ejecutemos la simulación:

```
prefix = 'ising'
input_file = pyalps.writeInputFiles(prefix,parms)
res = pyalps.runApplication('sparsediag', input_file)
# res = pyalps.runApplication('sparsediag', input_file, MPI=2, mpirun='mpirun')
data = pyalps.loadEigenstateMeasurements(pyalps.getResultFiles(prefix=prefix))
```

Al descomentar la penúltima línea y adaptar el número de trabajos y el nombre de su ejecutable mpirun (que por defecto será mpirun), podría hacer que ALPS use varias CPU simultáneamente.

Primero, extraeremos el nivel más bajo y el primer excitado para cada valor de L y los recopilaremos en un diccionario:

```
E0 = {}
E1 = {}
for Lsets in data:
    L = pyalps.flatten(Lsets)[0].props['L']
    allE = []
    for q in pyalps.flatten(Lsets):
        allE += list(q.y)
    allE = np.sort(allE)
    E0[L] = allE[0]
    E1[L] = allE[1]
```
        
El código anterior funciona porque sabemos que ALPS cargará las energías en listas agrupadas por simulación; los datos son por tanto una lista de listas donde tenemos distintas simulaciones en el nivel superior y distintos momentos debajo. Ahora reescalamos las energías según la ecuación dada arriba y recopilamos los datos en función del momento:

```
for q in pyalps.flatten(data):
    L = q.props['L']
    q.y = (q.y-E0[L])/(E1[L]-E0[L]) * (1./8.)
spectrum = pyalps.collectXY(data, 'TOTAL_MOMENTUM', 'Energy', foreach=['L'])
```
    
Para comparar, mostremos también los campos primarios y sus primeros descendientes:

```
for SD in [0.125, 1, 1+0.125, 2]:
    d = pyalps.DataSet()
    d.x = np.array([0,4])
    d.y = SD+0*d.x
    spectrum += [d]
```
        
Finalmente creamos el gráfico:

```
pyalps.plot.plot(spectrum)
plt.legend(prop={'size':8})
plt.xlabel("$k$")
plt.ylabel("E_0")
plt.xlim(-0.02, math.pi+0.02)
plt.show()
```

## Cadena de Heisenberg

Consideremos ahora un ejemplo más complicado: la cadena antiferromagnética de Heisenberg para grados de libertad de espín-1/2, descrita por
$$
H = \sum_{\langle i,j \rangle} \mathbf{S}^i \cdot \mathbf{S}^j
$$

La teoría crítica para este modelo tiene una carga central $c=1$ con campos primarios 0, 0.5 y 1. A diferencia del modelo de Ising, las correcciones de tamaño finito se anulan solo logarítmicamente y por tanto son bastante pronunciadas en los tamaños de sistema que alcanzamos. <a href="../codes/ed-04-criticality/parm_heisenberg" download>El archivo de parámetros</a> se puede encontrar aquí. <a href="../codes/ed-04-criticality/heisenberg.py" download>El archivo de Python</a> es análogo a la descripción anterior y por tanto no se describirá aquí en detalle. La diferencia principal es que podemos aprovechar la simetría U(1) del sistema; también ejecutamos las simulaciones solo para el sector $S_z = 0$, ya que este contiene todos los estados relevantes.

Al examinar el espectro, intente identificar los distintos campos de escala y cómo se aproximan a los valores correctos a medida que aumenta el tamaño del sistema. Notará que esto es bastante difícil. Para una discusión detallada del sistema, consulte [I Affleck et al 1989 J. Phys. A: Math. Gen. 22 511](https://iopscience.iop.org/article/10.1088/0305-4470/22/5/015) y el próximo tutorial.

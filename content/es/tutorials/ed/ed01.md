En este tutorial aprenderemos a usar el programa de diagonalización dispersa `sparsediag`, que encuentra los autoestados más bajos de un hamiltoniano cuántico mediante el algoritmo iterativo de Lanczos, y cómo leer observables arbitrarios sobre los autoestados resultantes.

## Medidas en una cadena de Heisenberg unidimensional

Como primer ejemplo consideramos la cadena de Heisenberg antiferromagnética isótropa para espín S=1,

$$
H = J \sum_{\langle i,j \rangle} \mathbf{S}_i \cdot \mathbf{S}_j , \qquad J>0 ,
$$

donde la suma recorre los enlaces a primeros vecinos de una cadena periódica. Este modelo es el ejemplo paradigmático de un antiferromagneto de espín entero: Haldane predijo, y desde entonces los cálculos numéricos han confirmado, que tiene un estado fundamental único y con gap, con correlaciones de espín que decaen exponencialmente y un orden antiferromagnético de corto alcance con un pico en el momento $q=\pi$ ([F.D.M. Haldane, Phys. Rev. Lett. 50, 1153 (1983)](https://doi.org/10.1103/PhysRevLett.50.1153)). La diagonalización exacta nos da acceso directo a la función de onda del estado fundamental de una cadena finita, a partir de la cual podemos calcular exactamente cualquier función de correlación o factor de estructura —este tutorial muestra cómo solicitar tales medidas a `sparsediag`. El gap de tamaño finito en sí mismo es el tema del siguiente tutorial, [ED-02](../ed02).

### Parámetros

| Parámetro | Significado | Valor |
|---|---|---|
| `LATTICE` | red periódica 1D incorporada | `chain lattice` |
| `MODEL` | modelo de espín cuántico | `spin` |
| `local_S` | número cuántico de espín por sitio | 1 |
| `J` | acoplamiento de Heisenberg a primeros vecinos | 1 |
| `L` | número de sitios | 4 |
| `CONSERVED_QUANTUMNUMBERS` | simetrías usadas para diagonalizar el hamiltoniano en bloques | `Sz` |
| `MEASURE_CORRELATIONS[...]` | solicita $\langle S^z_iS^z_j\rangle$ y $\langle S^+_iS^-_j\rangle$ en cada autoestado calculado | ver abajo |
| `MEASURE_STRUCTURE_FACTOR[...]` | solicita la transformada de Fourier de una función de correlación | $S^z$ |

### Red

La `chain lattice` es una de las geometrías incorporadas en la [biblioteca de redes de ALPS](../../../documentation/intro/latticehowtos); coloca `L` sitios en un anillo conectados por enlaces periódicos a primeros vecinos de intensidad `J`:

```
      J     J     J
  o-------o-------o-------o
  0       1       2       3
  |___________________________|
              J   (bond 3-0, periodic)
```

### Método

La cadena S=1 con L=4 sitios tiene un espacio de Hilbert de dimensión $3^4=81$, que se divide en un sector $S_z=0$ de dimensión 19 al usar el número cuántico `Sz`. Dado que solo necesitamos el estado fundamental (y, para el factor de estructura y las correlaciones de abajo, ningún otro autoestado), el algoritmo iterativo de Lanczos implementado por `sparsediag` es la elección natural: converge al autopar más bajo de un hamiltoniano disperso en unas pocas multiplicaciones matriz-vector, sin necesidad de construir ni diagonalizar nunca la matriz completa de $81\times 81$.

### Usando la línea de comandos

El archivo de parámetros <a href="../codes/ed-01-sparsediag/parm1a" download>`parm1a`</a> configura una diagonalización exacta de la cadena cuántica S=1 con 4 sitios:

```
MODEL="spin"
LATTICE="chain lattice"
CONSERVED_QUANTUMNUMBERS="Sz"
MEASURE_STRUCTURE_FACTOR[Structure Factor Sz]=Sz
MEASURE_CORRELATIONS[Diagonal spin correlations]=Sz
MEASURE_CORRELATIONS[Offdiagonal spin correlations]="Splus:Sminus"
local_S=1
J=1
{L=4;}
```
    
Lo nuevo aquí en comparación con otros códigos son los parámetros de medida que especifican qué promedios de operadores, valores locales, correlaciones y factores de estructura deben medirse. Hay más detalles sobre estas medidas personalizadas disponibles aquí.
Usando la secuencia estándar de comandos, primero puede convertir los parámetros de entrada a XML y luego ejecutar la aplicación `sparsediag` (tenga en cuenta que encontrará estos ejecutables en el directorio `bin` de su instalación de ALPS; puede agregar este directorio a su PATH para facilitar el trabajo con ALPS):

```
parameter2xml parm1a
sparsediag --write-xml parm1a.in.xml
```
    
Los autovalores y autoestados más bajos se calculan en cada sector (Sz,P), donde P denota el momento total. El archivo de salida `parm1a.task1.out.xml` contiene todas las cantidades calculadas y puede visualizarse con un navegador de internet estándar. En nuestro caso, el estado fundamental se encuentra en el sector Sz=0, P=0. Las correlaciones de espín diagonales correspondientes, mostradas en el archivo XML, se ven así

```
Diagonal spin correlations[( 0 ) -- ( 0 )]    (0.666667,0)
Diagonal spin correlations[( 0 ) -- ( 1 )]    (-0.5,0)
Diagonal spin correlations[( 0 ) -- ( 2 )]    (0.333333,0)
Diagonal spin correlations[( 0 ) -- ( 3 )]    (-0.5,0)
```
    
Los números \[( a ) -- ( b )\] en los corchetes anteriores se refieren a los índices de sitio, es decir, Sz(a)\*Sz(b). En la columna derecha se puede leer el valor (complejo) de la función de correlación.
La salida para el factor de estructura Sz de este estado se ve así

```
Structure Factor Sz[( 0 )]                    5.551115123125783e-17
Structure Factor Sz[( 1.570796326794897 )]    0.333333333333333
Structure Factor Sz[( 3.141592653589793 )]    2
Structure Factor Sz[( -1.5707963267948966 )]    0.3333333333333329
```

donde el número entre corchetes \[(q)\] denota el número de onda.
Se puede limitar explícitamente el sector Sz agregando la siguiente línea al archivo de parámetros:

```
Sz_total=0
```
    
### Usando Python

Para configurar y ejecutar la simulación en Python usamos el script <a href="../codes/ed-01-sparsediag/tutorial1a.py" download>`tutorial1a.py`</a>. Las primeras partes de este script importan el módulo requerido `pyalps`, preparan los archivos de entrada como una lista de diccionarios de Python, escriben los archivos de entrada y ejecutan la aplicación

```
import pyalps
parms = [{ 
        'LATTICE'                   : "chain lattice", 
        'MODEL'                     : "spin",
        'local_S'                   : 1,
        'J'                         : 1,
        'L'                         : 4,
        'CONSERVED_QUANTUMNUMBERS'  : 'Sz',
        'MEASURE_STRUCTURE_FACTOR[Structure Factor Sz]'       : 'Sz',
        'MEASURE_CORRELATIONS[Diagonal spin correlations]='   : 'Sz',
        'MEASURE_CORRELATIONS[Offdiagonal spin correlations]' : 'Splus:Sminus'
    }]

input_file = pyalps.writeInputFiles('parm1a',parms)
res = pyalps.runApplication('sparsediag',input_file)
```
    
Ahora puede lanzar su Python para ejecutar el script `tutorial1a.py`. Tendremos los mismos archivos de salida que en la versión de línea de comandos.
A continuación cargamos las medidas de cada uno de los autoestados calculados:

```
data = pyalps.loadEigenstateMeasurements(pyalps.getResultFiles(prefix='parm1a'))
```

y luego imprimimos los resultados solo para el estado fundamental:

```
for sector in data[0]:
    print '\nSector with Sz =', sector[0].props['Sz'], 
    print 'and k =', sector[0].props['TOTAL_MOMENTUM']
    for s in sector:
        if pyalps.size(s.y[0])==1:
            print s.props['observable'], ' : ', s.y[0]
        else:
            for (x,y) in zip(s.x,s.y[0]):
                print  s.props['observable'], '(', x, ') : ', y
```

## Resumen

Para la cadena S=1 con L=4, el estado fundamental se encuentra en el sector $(S_z,P)=(0,0)$ y muestra correlaciones diagonales antiferromagnéticas de corto alcance que alternan de signo con la distancia y decaen con la separación, así como un factor de estructura estático fuertemente concentrado en $q=\pi$ (valor 2, frente a $\approx 0.33$ en los momentos vecinos) —la firma esperada del orden de tipo Néel de corto alcance de una cadena de Haldane con gap, ya visible en este sistema tan pequeño.

## Preguntas

- ¿Por qué `Diagonal spin correlations[( 0 ) -- ( b )]` alterna de signo a medida que $b$ aumenta de 0 a 3?
- El pico del factor de estructura en $q=\pi$ tiene un valor finito (2) en lugar de divergir. ¿Esperaría que este pico crezca, disminuya o se mantenga igual al aumentar $L$? (Compárelo con el gap de tamaño finito que calculará en [ED-02](../ed02).)
- Añada `MEASURE_AVERAGE[Energy per bond]=bond_energy` (o lea directamente la energía total) y compruebe que es consistente con las correlaciones diagonales y no diagonales impresas arriba, usando $\mathbf{S}_i\cdot\mathbf{S}_j = S^z_iS^z_j + \tfrac12(S^+_iS^-_j+S^-_iS^+_j)$.

En este tutorial usaremos la aplicación `fulldiag` para calcular el espectro *completo* de sistemas de espín cuánticos pequeños, y lo utilizaremos para calcular cantidades termodinámicas exactas a temperatura finita —energía, entropía, calor específico y susceptibilidad— como una suma ponderada sobre todos los autoestados, completamente libre del ruido estadístico del Monte Carlo cuántico o de los errores de truncamiento del DMRG. Esto complementa los tutoriales anteriores, que usaban `sparsediag` para calcular solo los primeros autoestados más bajos de un hamiltoniano: aquí necesitamos el espectro *completo*, porque cada estado, por muy alta que sea su energía, tiene un peso de Boltzmann $e^{-E_n/T}$ a cualquier temperatura finita.

## Termodinámica de modelos de espín unidimensionales

### La cadena de Heisenberg de espín 1

Nuestra primera simulación será para una cadena cuántica antiferromagnética de Heisenberg de espín 1,
$$
H = J \sum_{\langle i,j \rangle} \mathbf{S}_i \cdot \mathbf{S}_j , \qquad J=1 ,
$$
en una cadena periódica, cuya termodinámica de cadena finita fue estudiada sistemáticamente por primera vez por [J.C. Bonner and M.E. Fisher, Phys. Rev. 135, A640 (1964)](https://doi.org/10.1103/PhysRev.135.A640).

#### Parámetros

| Parámetro | Significado | Valor |
|---|---|---|
| `LATTICE` | cadena periódica | `chain lattice` |
| `MODEL` | modelo de espín cuántico | `spin` |
| `local_S` | número cuántico de espín por sitio | 1 |
| `J` | acoplamiento a primeros vecinos | 1 |
| `L` | longitud de la cadena | 8 |
| `CONSERVED_QUANTUMNUMBERS` | simetría usada para diagonalizar $H$ en bloques | `Sz` |

#### Red

La misma `chain lattice` periódica usada en toda esta serie de tutoriales (véase la [biblioteca de redes de ALPS](../../../documentation/intro/latticehowtos)):

```
    J     J     J     J     J     J     J
o-------o-------o-------o-------o-------o-------o-------o
0       1       2       3       4       5       6       7
|_______________________________________________________|
                      J   (bond 7-0, periodic)
```

#### Método

Las cantidades termodinámicas requieren sumar pesos de Boltzmann sobre el espectro *completo*, no solo sobre el estado fundamental, por lo que se usa `fulldiag` en lugar de `sparsediag`. El espacio de Hilbert completo de esta cadena S=1, L=8 tiene dimensión $3^8=6561$; usando `CONSERVED_QUANTUMNUMBERS=Sz` se divide en 17 bloques mucho más pequeños (de a lo sumo $1107$ estados, el sector $S_z=0$), cada uno de los cuales se diagonaliza directamente, lo que es a la vez más rápido y suficiente para reconstruir la función de partición exacta a cualquier temperatura.

#### Usando la línea de comandos

El archivo de parámetros `parm6a` configura una diagonalización completa para un antiferromagneto cuántico de Heisenberg de espín 1 en una cadena unidimensional con 8 sitios.

```
LATTICE="chain lattice"
MODEL="spin"
local_S = 1
J       = 1
CONSERVED_QUANTUMNUMBERS="Sz"
{L = 8}
```

El parámetro CONSERVED_QUANTUMNUMBERS ayuda a `fulldiag` a dividir el espacio de Hilbert en subespacios invariantes y a diagonalizarlos por separado.
Usando la secuencia estándar de comandos, primero puede convertir los parámetros de entrada a XML y luego calcular el espectro completo de este hamiltoniano cuántico usando `fulldiag`:

```
parameter2xml parm6a
fulldiag --write-xml parm6a.in.xml
```
    
El archivo de salida ahora contiene resultados para todos los autovectores, y puede producir archivos de gráfico XML para los observables termodinámicos y magnéticos usando `fulldiag_evaluate`:

```
fulldiag_evaluate --T_MIN 0.1 --T_MAX 10 --DELTA_T  0.1  parm6a.task1.out.xml
```

Esto generará los siguientes archivos de gráfico XML:

```
parm6a.task1.plot.energy.xml
parm6a.task1.plot.free_energy.xml
parm6a.task1.plot.entropy.xml
parm6a.task1.plot.specific_heat.xml
parm6a.task1.plot.uniform_susceptibility.xml
parm6a.task1.plot.magnetization.xml
```

Para extraer los resultados calculados de los archivos de gráfico XML generados por `fulldiag_evaluate`, puede usar la herramienta `plot2text`, y luego visualizar estos datos con su herramienta de graficación favorita. Por ejemplo, para extraer los datos de la densidad de energía frente a la temperatura, use

```
plot2text parm6a.task1.plot.energy.xml
```

De forma similar, puede extraer los datos de los otros archivos de gráfico XML.
Si Grace es su herramienta de graficación favorita, también puede generar directamente un archivo de proyecto de Grace a partir del archivo de gráfico XML usando la herramienta plot2xmgr. Por ejemplo, para generar un archivo de proyecto de Grace de la energía frente a la temperatura, use

```
plot2xmgr parm6a.task1.plot.energy.xml > energy.agr
```

De manera similar, la herramienta `plot2gp` produce scripts de Gnuplot y `plot2text` convierte el archivo a texto plano. Sin embargo, el método preferido para el análisis y la graficación de datos es usar Python.

#### Usando Python
Para configurar y ejecutar la simulación en Python usamos el script `tutorial6a.py`. Las primeras partes de este script importan los módulos requeridos y luego preparan los archivos de entrada como una lista de diccionarios de Python, y a continuación ejecutan la simulación

```
import pyalps
import matplotlib.pyplot as plt
import pyalps.plot
import numpy as np
parms = [{ 
        'LATTICE'                   : "chain lattice", 
        'MODEL'                     : "spin",
        'CONSERVED_QUANTUMNUMBERS'  : 'Sz',
        'local_S'                   : 1,
        'J'                         : 1,
        'L'                         : 8
    }]

input_file = pyalps.writeInputFiles('parm6a',parms)
res = pyalps.runApplication('fulldiag',input_file)
```

A continuación ejecutamos el programa de evaluación sobre todos los archivos de salida

```
data = pyalps.evaluateFulldiagVersusT(pyalps.getResultFiles(prefix='parm6a'),DELTA_T=0.1, T_MIN=0.1, T_MAX=10.0)
```

y finalmente mostramos todos los gráficos:

```
for s in pyalps.flatten(data):
    plt.figure()
    plt.title("Antiferromagnetic Heisenberg chain")
    pyalps.plot.plot(s)
    plt.show()
```

#### Datos de salida

Diagonalizando el mismo hamiltoniano de forma independiente se obtienen las siguientes cantidades termodinámicas por sitio (energía $E/L$, calor específico $C/L$, susceptibilidad uniforme $\chi/L$):

| $T/J$ | $E/L$ | $C/L$ | $\chi/L$ |
|---|---|---|---|
| 0.1 | -1.417 | 0.034 | 0.0066 |
| 0.2 | -1.407 | 0.138 | 0.0567 |
| 0.5 | -1.333 | 0.401 | 0.1276 |
| 1.0 | -1.064 | 0.559 | 0.1693 |
| 2.0 | -0.653 | 0.278 | 0.1639 |
| 5.0 | -0.274 | 0.055 | 0.1010 |

El calor específico tiene un máximo amplio alrededor de $T\approx J$, mientras que la susceptibilidad presenta un pico cerca de $T\approx1$–$1.5\,J$ y luego cae bruscamente hacia $T=0$ —el remanente de tamaño finito del comportamiento activado exponencialmente $\chi\sim e^{-\Delta/T}$ esperado a partir del gap de Haldane $\Delta\approx0.41\,J$ calculado en [ED-02](../ed02).

### La escalera de Heisenberg de espín 1/2

Ahora cambiamos el modelo a una escalera de dos patas,
$$
H = J_0 \sum_{\text{legs}} \mathbf{S}_i \cdot \mathbf{S}_j + J_1 \sum_{\text{rungs}} \mathbf{S}_i \cdot \mathbf{S}_j ,
$$
de longitud L=6 (es decir, 6 peldaños, 12 sitios), fijando el parámetro LATTICE en "ladder" y los acoplamientos J0 y J1 en 1 —la misma red usada en [ED-03](../ed03):

```
      o---J0---o---J0---o---J0---o---J0---o---J0---o     leg 0
      |        |        |        |        |        |
      J1       J1       J1       J1       J1       J1
      |        |        |        |        |        |
      o---J0---o---J0---o---J0---o---J0---o---J0---o     leg 1
```

Los parámetros están en `parm6b`, el script de Python en `tutorial6b.py`. Ejecútelo exactamente igual que `parm6a` arriba:

```
parameter2xml parm6b
fulldiag --write-xml parm6b.in.xml
fulldiag_evaluate --T_MIN 0.1 --T_MAX 10 --DELTA_T 0.1 parm6b.task1.out.xml
```

`parm6b` es idéntico a `parm6a` salvo que usa `LATTICE="ladder"`, `J0=1`, `J1=1`, y `{L = 6}` en lugar de los parámetros de la cadena.

#### Datos de salida

Diagonalizando esta escalera L=6 de forma independiente (con condiciones de frontera abiertas en las patas) se obtiene un gap singlete-triplete de tamaño finito $\Delta\approx0.68\,J$, algo por encima del valor aceptado para la escalera infinita, $\approx 0.5\,J$, citado en la pista de abajo —un recordatorio de que un sistema de 6 peldaños todavía está bastante lejos del límite termodinámico para un estado con gap. La termodinámica por sitio correspondiente:

| $T/J$ | $E/L$ | $C/L$ | $\chi/L$ |
|---|---|---|---|
| 0.1 | -0.550 | 0.013 | 0.0019 |
| 0.2 | -0.544 | 0.126 | 0.0294 |
| 0.5 | -0.451 | 0.424 | 0.1065 |
| 0.7 | -0.368 | 0.382 | 0.1204 |
| 1.0 | -0.274 | 0.252 | 0.1189 |
| 2.0 | -0.137 | 0.072 | 0.0876 |

(¡Pruébelo! Puede que descubra que en realidad no es tan fácil hacerlo mejor con QMC que con la diagonalización completa para los ejemplos anteriores.)

La diagonalización exacta sigue siendo, sin duda, el método de elección bajo ciertas condiciones. En primer lugar, si el sistema es intrínsecamente finito (y posiblemente pequeño), la diagonalización completa proporciona el resultado exacto de inmediato. En segundo lugar, la diagonalización exacta no sufre del problema del signo, por lo que también puede usarse para modelos fermiónicos o frustrados sin ninguna restricción adicional. Ambas condiciones se cumplen simultáneamente en los modelos de moléculas magnéticas que se discutirán en la segunda parte.

## Termodinámica de moléculas magnéticas

Ahora usaremos `fulldiag` para simular la termodinámica exacta de pequeños clústeres de espín, calculando su espectro completo.

### Dos dímeros acoplados

#### Configurando una red/grafo y modelo personalizados

Considere el siguiente sistema de dos dímeros acoplados: un dímero "superior" de dos espines $S_0$ acoplados por $J_0$, un dímero "inferior" de dos espines $S_1$ también acoplados por $J_0$, y cada espín del dímero superior acoplado a cada espín del dímero inferior por $J_1$,

$$
H = J_0\left(\mathbf{S}_1\cdot\mathbf{S}_2 + \mathbf{S}_3\cdot\mathbf{S}_4\right) + J_1\left(\mathbf{S}_1\cdot\mathbf{S}_3+\mathbf{S}_1\cdot\mathbf{S}_4+\mathbf{S}_2\cdot\mathbf{S}_3+\mathbf{S}_2\cdot\mathbf{S}_4\right) - h\sum_{i=1}^4 S^z_i .
$$

```
        J0
   1 ---------- 2        (upper dimer, spin S0)
   |  \        /  |
   |   \  J1  /   |
   J1    \  /    J1
   |     /  \     |
   |   /      \   |
   3 ---------- 4        (lower dimer, spin S1)
        J0
```

Este es precisamente el grafo codificado a continuación: los sitios 1, 2 (tipo 0) forman el dímero superior, los sitios 3, 4 (tipo 1) el dímero inferior, las aristas de tipo 0 son los enlaces intra-dímero ($J_0$) y las aristas de tipo 1 son los cuatro enlaces inter-dímero ($J_1$).

Primero, necesitamos un grafo que represente este problema. Esto se define mediante la siguiente entrada en el archivo `dd-graph.xml`:

```
<LATTICES>
<GRAPH name="double dimer" vertices="4">
<VERTEX id="1" type="0"></VERTEX>
<VERTEX id="2" type="0"></VERTEX>
<VERTEX id="3" type="1"></VERTEX>
<VERTEX id="4" type="1"></VERTEX>
<EDGE type="0" source="1" target="2"/>
<EDGE type="0" source="3" target="4"/>
<EDGE type="1" source="1" target="3"/>
<EDGE type="1" source="1" target="4"/>
<EDGE type="1" source="2" target="3"/>
<EDGE type="1" source="2" target="4"/>
</GRAPH>
</LATTICES>
```

Nota: el archivo debe usar XML puro sin declaración `<?xml?>` ni cabecera `<!DOCTYPE>` —el analizador de ALPS no admite eso.
    
Luego también queremos asignar distintos intercambios de Heisenberg J0 y J1 a las aristas de tipo 0 y 1, respectivamente. Esto se logra mediante la siguiente entrada en el archivo `model-dspin.xml`:

```
<MODELS>
<SITEBASIS name="spin">
  <PARAMETER name="local_spin" default="local_S"/>
  <PARAMETER name="local_S" default="1/2"/>
  <QUANTUMNUMBER name="S" min="local_spin" max="local_spin"/>
  <QUANTUMNUMBER name="Sz" min="-S" max="S"/>
  <OPERATOR name="Splus" matrixelement="sqrt(S*(S+1)-Sz*(Sz+1))">
    <CHANGE quantumnumber="Sz" change="1"/>
  </OPERATOR>
  <OPERATOR name="Sminus" matrixelement="sqrt(S*(S+1)-Sz*(Sz-1))">
    <CHANGE quantumnumber="Sz" change="-1"/>
  </OPERATOR>
  <OPERATOR name="Sz" matrixelement="Sz"/>
</SITEBASIS>
<BASIS name="spin">
  <SITEBASIS ref="spin">
    <PARAMETER name="local_spin" value="local_S#"/>
    <PARAMETER name="local_S#" value="1/2"/>
  </SITEBASIS>
  <CONSTRAINT quantumnumber="Sz" value="Sz_total"/>
</BASIS>
<HAMILTONIAN name="dimerized spin">
<PARAMETER name="J" default="1"/>
<PARAMETER name="h" default="0"/>
<BASIS ref="spin"/>
<SITETERM site="i">
<PARAMETER name="h#" default="h"/>
    -h#*Sz(i)
</SITETERM>
<BONDTERM source="i" target="j">
<PARAMETER name="J#" default="J"/>
    J#*Sz(i)*Sz(j)+J#/2*(Splus(i)*Sminus(j)+Sminus(i)*Splus(j))
</BONDTERM>
</HAMILTONIAN>
</MODELS>
```

Nota: este archivo también debe usar XML puro sin declaración `<?xml?>` ni cabecera `<!DOCTYPE>`, y debe incluir las definiciones `<SITEBASIS>` y `<BASIS>` —sin ellas, fulldiag no puede construir el espacio de Hilbert.
    
Nótese que en realidad no necesitamos esta definición, ya que el hamiltoniano "spin" del archivo `models.xml` por defecto ya contiene definiciones adecuadas. Sin embargo, el ejemplo anterior ilustra cómo asignar automáticamente constantes de intercambio Jn a un enlace de tipo n, usando el símbolo de almohadilla (\#).

De paso, también hemos asignado tipos distintos a los vértices 1, 2 y 3, 4. Por lo tanto, podemos asignar espines locales distintos al dímero superior e inferior especificando los valores correspondientes mediante local_S0 y local_S1, respectivamente.

Ahora calcularemos la curva de magnetización para espines locales $S_0=1$ (dímero superior) y $S_1=1/2$ (dímero inferior), $J_0=1$, $J_1=0.4$, a una temperatura $T=0.02$.

#### Usando la línea de comandos

El archivo de parámetros `parm6c` configura la simulación:

```
LATTICE="double dimer"
MODEL="dimerized spin"
LATTICE_LIBRARY="dd-graph.xml"
MODEL_LIBRARY="model-dspin.xml"
local_S0=1
local_S1=1/2
J0      = 1
J1      = 0.4
h       = 0 
CONSERVED_QUANTUMNUMBERS="Sz"
{T = 0.02}
```

Nótense los nuevos parámetros LATTICE_LIBRARY y MODEL_LIBRARY, que apuntan a los archivos que contienen nuestra red y modelo personalizados.
El cálculo se realiza con la siguiente secuencia de comandos:

```
parameter2xml parm6c
fulldiag --write-xml parm6c.in.xml
fulldiag_evaluate --H_MIN 0 --H_MAX 4 --DELTA_H 0.025 --versus h parm6c.task1.out.xml
```

Nótese que aquí hemos usado `fulldiag_evaluate` con argumentos de línea de comandos para especificar un rango de campo magnético. En particular, el argumento de línea de comandos --versus h coloca el campo magnético en el eje x en lugar de la temperatura, como en los ejemplos anteriores.

#### Método

Con solo $(2\cdot1+1)^2\times(2\cdot\tfrac12+1)^2=36$ estados base, este sistema es trivial para `fulldiag` —el objetivo de este ejemplo no es la dificultad computacional, sino mostrar cómo montar, enteramente a partir de archivos de parámetros, un grafo de red y un hamiltoniano completamente personalizados, del tipo que se necesita para datos reales de moléculas magnéticas.

#### Datos de salida

A la baja temperatura $T=0.02\,J_0$ usada aquí, el espín total $S_{\text{total}}$ del estado fundamental aumenta en pasos unitarios a medida que se incrementa $h$, y cada paso ocurre donde se cruzan dos niveles del espectro analítico de abajo:

| $h/J_0$ | $\langle S^z_{\text{total}}\rangle$ | sector del estado fundamental |
|---|---|---|
| 0.0 – 0.8 | 0.00 | $S_{\text{total}}=0$ |
| 1.0 | 0.67 | cruce ($S_{\text{total}}=0\to1$) |
| 1.2 | 1.00 | $S_{\text{total}}=1$ |
| 1.5 | 1.99 | cruce ($S_{\text{total}}=1\to2$) |
| 1.8 – 2.2 | 2.00 | $S_{\text{total}}=2$ |
| 2.5 | 2.99 | cruce ($S_{\text{total}}=2\to3$) |
| 3.0 – 4.0 | 3.00 | $S_{\text{total}}=3$ |

Los tres campos de cruce se obtienen directamente de las energías analíticas dadas en la pista de abajo: igualando la energía más baja de $S_{\text{total}}=n$ (en $S^z=n$) con la de $S_{\text{total}}=n+1$ (en $S^z=n+1$) se obtiene $h_1=1.0$, $h_2=1.4$ y $h_3=2.4$, en unidades de $J_0$ —una escalera de magnetización con plateaus en $S^z_{\text{total}}=0,1,2,3$, redondeada solo por la temperatura pequeña pero no nula.

### El complejo molecular $V_{15}$

El último ejemplo es un modelo para el nanomagneto molecular $\mathrm{V}_{15}$, en el que 15 iones $\mathrm{V}^{4+}$ ($S=1/2$ cada uno) se sitúan en dos hexágonos que encierran un triángulo central. Su magnetismo de baja energía está dominado por un intercambio antiferromagnético fuerte alrededor de los dos hexágonos y un intercambio más débil que los conecta con el triángulo central, de modo que a baja temperatura los 15 espines se comportan efectivamente como un triángulo frustrado de espín 1/2: dos dobletes de baja energía más un cuadruplete, bien separados del resto del espectro ([G. Chaboussant et al., Europhys. Lett. 59, 291 (2002)](https://doi.org/10.1209/epl/i2002-00240-x)).

```
        1---2                 outer hexagon (6 spins)
       /     \
      6       3
       \     /
        5---4
       /     \
      o       o                inner hexagon (6 spins),
       \     /                 rotated relative to the outer one
        o---o
          |
        triangle (3 spins)    weakly coupled to both hexagons
```

Para las simulaciones hacemos la suposición simplificadora de un intercambio de Heisenberg $J$ igual a lo largo de todas las aristas de este grafo. Configurar los archivos de red y modelo personalizados sigue exactamente la misma receta que el ejemplo de los dímeros acoplados de arriba —un `<GRAPH>` con 15 vértices y la lista `<EDGE>` correspondiente para los enlaces de los hexágonos y del triángulo, y el mismo hamiltoniano `dimerized spin` (o el `spin` incorporado). El grafo asociado está definido en `v15-graph.xml`. La ejecución de las simulaciones procede ahora como antes. Los parámetros están en `parm6d`, el script de Python en `tutorial6d.py`.

#### Método

El espacio de Hilbert completo de 15 sitios de espín 1/2 tiene dimensión $2^{15}=32768$; incluso resuelto en sectores $S_z$, `fulldiag` tiene que diagonalizar exactamente un bloque de varios miles de estados para obtener correctamente la termodinámica a baja temperatura, razón por la cual —como indica la nota de abajo— esta ejecución toma notablemente más tiempo que los ejemplos anteriores.

## Resumen

La diagonalización completa proporciona termodinámica numéricamente exacta para cualquier sistema de espín cuántico finito, desde cadenas y escaleras invariantes bajo traslación hasta moléculas magnéticas totalmente personalizadas y frustradas —el precio es el crecimiento exponencial del espacio de Hilbert con el tamaño del sistema, razón por la cual el método está confinado a algunas decenas de espines incluso explotando simetrías, en contraste con los sistemas mucho más grandes alcanzables con `sparsediag` para propiedades de estado fundamental y de baja energía únicamente.

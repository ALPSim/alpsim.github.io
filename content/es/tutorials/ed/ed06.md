En este tutorial usaremos la aplicación fulldiag para calcular el espectro completo de cadenas cuánticas y, a partir de él, cantidades termodinámicas.

## Termodinámica de modelos de espín unidimensionales

### La cadena de Heisenberg de espín-1

Nuestra primera simulación será para una cadena cuántica antiferromagnética de Heisenberg de espín-1.

#### Usando la línea de comandos

El archivo de parámetros `parm6a` configura una diagonalización completa para un antiferromagneto cuántico de Heisenberg de espín-1 en una cadena unidimensional con 8 sitios.

```
LATTICE="chain lattice"
MODEL="spin"
local_S = 1
J       = 1
CONSERVED_QUANTUMNUMBERS="Sz"
{L = 8}
```

El parámetro CONSERVED_QUANTUMNUMBERS ayuda a `fulldiag` a dividir el espacio de Hilbert en subespacios invariantes y diagonalizarlos por separado.
Usando la secuencia estándar de comandos, primero puede convertir los parámetros de entrada a XML y luego calcular el espectro completo de este hamiltoniano cuántico usando `fulldiag`:

```
parameter2xml parm6a
fulldiag --write-xml parm6a.in.xml
```
    
El archivo de salida ahora contiene resultados para todos los autovectores y puede producir archivos XML de gráficos para los observables termodinámicos y magnéticos usando `fulldiag_evaluate`:
Ahora puede producir archivos XML de gráficos para los observables termodinámicos y magnéticos usando `fulldiag_evaluate`:

```
fulldiag_evaluate --T_MIN 0.1 --T_MAX 10 --DELTA_T  0.1  parm6a.task1.out.xml
```

Esto generará los siguientes archivos XML de gráficos:

```
parm6a.task1.plot.energy.xml
parm6a.task1.plot.free_energy.xml
parm6a.task1.plot.entropy.xml
parm6a.task1.plot.specific_heat.xml
parm6a.task1.plot.uniform_susceptibility.xml
parm6a.task1.plot.magnetization.xml
```

Para extraer los resultados calculados de los archivos XML de gráficos generados por `qwl_evaluate`, puede usar la herramienta `plot2text`, y luego visualizar estos datos con su herramienta de gráficos favorita. Por ejemplo, para extraer los datos de la densidad de energía en función de la temperatura, use

```
plot2text parm6a.task1.plot.energy.xml
```

De manera similar, puede extraer los datos de los otros archivos XML de gráficos.
Cuando Grace es su herramienta de gráficos favorita, también puede generar directamente un archivo de proyecto Grace a partir del archivo XML de gráfico usando la herramienta plot2xmgr. Por ejemplo, para generar un archivo de proyecto Grace de la energía en función de la temperatura, use

```
plot2xmgr parm6a.task1.plot.energy.xml > energy.agr
```

De manera similar, la herramienta `plot2gp` produce scripts de Gnuplot y `plot2text` convierte el archivo a texto plano. Sin embargo, el método preferido para la evaluación de datos y la graficación es usar Python.

#### Usando Python
Para configurar y ejecutar la simulación en Python usamos el script `tutorial6a.py`. Las primeras partes de este script importan los módulos requeridos y luego preparan los archivos de entrada como una lista de diccionarios de Python, y luego ejecutan la simulación

```
import pyalps
import matplotlib.pyplot as plt
import pyalps.plot
import numpy as np
parms = [{ 
        'LATTICE'                   : "chain lattice", 
        'MODEL'                     : "spin",
        'CONSERVED_QUANTUMNUMBERS'  : 'Sz',
        'local_S'                   : 1,
        'J'                         : 1,
        'L'                         : 8
    }]

input_file = pyalps.writeInputFiles('parm6a',parms)
res = pyalps.runApplication('fulldiag',input_file)
```

A continuación ejecutamos el programa de evaluación sobre todos los archivos de salida

```
data = pyalps.evaluateFulldiagVersusT(pyalps.getResultFiles(prefix='parm6a'),DELTA_T=0.1, T_MIN=0.1, T_MAX=10.0)
```

y finalmente mostramos todos los gráficos:

```
for s in pyalps.flatten(data):
    plt.figure()
    plt.title("Antiferromagnetic Heisenberg chain")
    pyalps.plot.plot(s)
    plt.show()
```

**Observaciones:** Si está familiarizado con las simulaciones de Monte Carlo Cuántico (QMC), sabrá que se pueden tratar sistemas más grandes y por lo tanto obtener mejores aproximaciones al límite termodinámico. (¡Inténtelo! Puede descubrir que en realidad no es tan fácil mejorar con QMC respecto a la diagonalización completa para los ejemplos anteriores.)

La diagonalización exacta sigue siendo definitivamente el método de elección bajo ciertas condiciones. Primero, si el sistema es intrínsecamente finito (y posiblemente pequeño), la diagonalización completa produce el resultado exacto de inmediato. Segundo, la diagonalización exacta no sufre del problema del signo, por lo que también puede usarse para modelos fermiónicos o frustrados sin ninguna restricción adicional. Ambas condiciones se satisfacen simultáneamente en los modelos de moléculas magnéticas que se discutirán en la segunda parte.

## Termodinámica de moléculas magnéticas

Ahora usaremos `fulldiag` para simular la termodinámica exacta de pequeños clústeres de espín, calculando su espectro completo.

### Dos dímeros acoplados

#### Configurando una red/grafo y modelo personalizados

Consideremos el siguiente sistema de dos dímeros acoplados:

Primero, necesitamos un grafo que represente este problema. Esto se define mediante la siguiente entrada en el archivo `dd-graph.xml`:

```
<LATTICES>
<GRAPH name="double dimer" vertices="4">
<VERTEX id="1" type="0"></VERTEX>
<VERTEX id="2" type="0"></VERTEX>
<VERTEX id="3" type="1"></VERTEX>
<VERTEX id="4" type="1"></VERTEX>
<EDGE type="0" source="1" target="2"/>
<EDGE type="0" source="3" target="4"/>
<EDGE type="1" source="1" target="3"/>
<EDGE type="1" source="1" target="4"/>
<EDGE type="1" source="2" target="3"/>
<EDGE type="1" source="2" target="4"/>
</GRAPH>
</LATTICES>
```

Nota: el archivo debe usar XML puro, sin declaración `<?xml?>` ni encabezado `<!DOCTYPE>` — el analizador de ALPS no admite esos elementos.
    
Luego también queremos asignar diferentes intercambios de Heisenberg J0 y J1 a las aristas de tipo 0 y 1, respectivamente. Esto se logra mediante la siguiente entrada en el archivo `model-dspin.xml`:

```
<MODELS>
<SITEBASIS name="spin">
  <PARAMETER name="local_spin" default="local_S"/>
  <PARAMETER name="local_S" default="1/2"/>
  <QUANTUMNUMBER name="S" min="local_spin" max="local_spin"/>
  <QUANTUMNUMBER name="Sz" min="-S" max="S"/>
  <OPERATOR name="Splus" matrixelement="sqrt(S*(S+1)-Sz*(Sz+1))">
    <CHANGE quantumnumber="Sz" change="1"/>
  </OPERATOR>
  <OPERATOR name="Sminus" matrixelement="sqrt(S*(S+1)-Sz*(Sz-1))">
    <CHANGE quantumnumber="Sz" change="-1"/>
  </OPERATOR>
  <OPERATOR name="Sz" matrixelement="Sz"/>
</SITEBASIS>
<BASIS name="spin">
  <SITEBASIS ref="spin">
    <PARAMETER name="local_spin" value="local_S#"/>
    <PARAMETER name="local_S#" value="1/2"/>
  </SITEBASIS>
  <CONSTRAINT quantumnumber="Sz" value="Sz_total"/>
</BASIS>
<HAMILTONIAN name="dimerized spin">
<PARAMETER name="J" default="1"/>
<PARAMETER name="h" default="0"/>
<BASIS ref="spin"/>
<SITETERM site="i">
<PARAMETER name="h#" default="h"/>
    -h#*Sz(i)
</SITETERM>
<BONDTERM source="i" target="j">
<PARAMETER name="J#" default="J"/>
    J#*Sz(i)*Sz(j)+J#/2*(Splus(i)*Sminus(j)+Sminus(i)*Splus(j))
</BONDTERM>
</HAMILTONIAN>
</MODELS>
```

Nota: el archivo debe usar XML puro, sin declaración `<?xml?>` ni encabezado `<!DOCTYPE>`, y debe incluir las definiciones de `<SITEBASIS>` y `<BASIS>` —sin ellas, fulldiag no puede construir el espacio de Hilbert.
    
Nótese que en realidad no necesitamos esta definición, ya que el hamiltoniano "spin" en el archivo `models.xml` predeterminado ya contiene definiciones adecuadas. No obstante, el ejemplo anterior ilustra cómo asignar automáticamente constantes de intercambio Jn a una arista de tipo n, usando el símbolo de almohadilla (\#).

De paso, también hemos asignado tipos diferentes a los vértices 1,2 y 3,4. Por lo tanto, podemos asignar espines locales diferentes al dímero superior e inferior especificando los valores correspondientes mediante local_S0 y local_S1, respectivamente.

Ahora calcularemos la curva de magnetización para espines locales $S_0=1$ (dímero superior) y $S_1=1/2$ (dímero inferior), $J_0=1$, $J_1=0.4$ a una temperatura $T=0.02$.

#### Usando la línea de comandos

El archivo de parámetros `parm6c` configura la simulación:

```
LATTICE="double dimer"
MODEL="dimerized spin"
LATTICE_LIBRARY="dd-graph.xml"
MODEL_LIBRARY="model-dspin.xml"
local_S0=1
local_S1=1/2
J0      = 1
J1      = 0.4
h       = 0 
CONSERVED_QUANTUMNUMBERS="Sz"
{T = 0.02}
```

Note los nuevos parámetros LATTICE_LIBRARY y MODEL_LIBRARY, que apuntan a los archivos que contienen nuestra red y modelo personalizados.
El cálculo se realiza con la siguiente secuencia de comandos:

```
parameter2xml parm6c
fulldiag --write-xml parm6c.in.xml
fulldiag_evaluate --H_MIN 0 --H_MAX 4 --DELTA_H 0.025 --versus h parm6c.task1.out.xml
```

Nótese que aquí hemos usado `fulldiag_evaluate` con argumentos de línea de comandos para especificar un rango de campo magnético. En particular, el argumento de línea de comandos --versus h coloca el campo magnético en el eje x en lugar de la temperatura, como en los ejemplos anteriores.

**Pista:** El espectro de dos dímeros acoplados $S_0=1$ y $S_1=1/2$ se puede encontrar analíticamente. Las energías son (con algunas degeneraciones):
- $-11J_0/4$, $-3J_0/4-2J_1$ para espín total $S_{\text{total}}=0$;

- $5J_0/4+2J_1$ para $S_{\text{total}}=3$.

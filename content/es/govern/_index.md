## Misión de ALPS

ALPS tiene como objetivo albergar y distribuir software para la simulación de sistemas cuánticos correlacionados.

Actualmente estamos reestructurando la gobernanza de ALPS. El trabajo sobre la estructura futura se realizará en nuestros talleres y se anunciará aquí. Si estás interesado en involucrarte, por favor házselo saber al liderazgo actual.

## Documentos de Gobernanza de ALPS

### Resumen 

El paquete de software ALPS (Applications and Libraries for Physics Simulations) proporciona un ecosistema de código abierto de algoritmos con aplicaciones en materia condensada, computación cuántica, información cuántica y campos relacionados. Este proyecto apoya a la comunidad científica de usuarios proporcionando una infraestructura de código abierto mantenible y sostenible para ALPS, junto con esfuerzos de construcción de comunidad.
ALPS está gobernado por [un consejo autoelegido](#comité-directivo-comunitario-de-alps). Los lanzamientos de ALPS estarán bajo la licencia de código abierto MIT. Para involucrarte en el desarrollo del proyecto, [envía un correo a un miembro del consejo directivo](#comité-directivo-comunitario-de-alps).

### Roles y Responsabilidades 

ALPS adopta una estructura de gobernanza jerárquica compartida para cada uno de sus roles técnicos.
Una comunidad de desarrolladores/colaboradores presenta issues, hace pull requests y contribuye al proyecto a través de [GitHub](https://github.com/ALPSim/ALPS).
Un conjunto de mantenedores, al menos uno por cada código de simulación, impulsa cada contribución al proyecto ALPS.
Son certificados por mantenedores principales (core maintainers), quienes imponen requisitos de compromiso y responden a los issues de la comunidad.
El consejo directivo impulsa la dirección general del proyecto, establece los requisitos de compromiso del código y toma las decisiones de deprecación.
El comité asesor externo hace recomendaciones respecto a la dirección y los enfoques para la participación comunitaria.

#### Mantenedores:

Cada código tiene un grupo de mantenedores que usa GitHub para enviar solicitudes de cambio. Los grupos de mantenedores son responsables de hacer pull requests en GitHub y de cambiar el alcance de su código. Cada grupo de mantenedores es responsable de seleccionar a uno o más miembros para dedicar tiempo como mantenedor principal (core maintainer). El consejo directivo decide el alcance de ese compromiso.

Una lista actual de mantenedores y sus áreas de responsabilidad está disponible en la página de [Mantenedores del Código](maintainers).

#### Mantenedores Principales (Core Maintainers):
    
Los mantenedores principales tienen dos roles primarios. 1) Responden a los issues de la comunidad. Esto incluye correcciones de bugs y pull requests. 2) Certifican las solicitudes de cambios y los pull requests hechos por los mantenedores. Estas certificaciones incluyen validar ejecuciones, compilaciones y verificación de bugs.

#### Consejo Directivo:
    
El [consejo directivo](#comité-directivo-comunitario-de-alps) dirige la trayectoria general del proyecto ALPS con el asesoramiento del [comité asesor externo](#comité-asesor-externo-de-alps). Las tareas del consejo incluyen:

- Nominar, confirmar y remover mantenedores y mantenedores principales
- Establecer hojas de ruta para los códigos, bibliotecas y dependencias que usarán los mantenedores
- Elegir y remover miembros del consejo y del comité asesor
- Liderar el proceso de publicación de los artículos de lanzamiento de ALPS

#### Comité Asesor Externo:

El comité asesor externo recomendará:

- Direcciones generales para el proyecto ALPS
- Direcciones para el crecimiento y mantenimiento de la comunidad

### Soporte 

- Para reportar un bug o solicitar una funcionalidad, por favor visita nuestro [repositorio](https://github.com/ALPSim/ALPS/issues) de GitHub.

- Para obtener ayuda con el uso de ALPS, por favor visita nuestro foro de usuarios en [Discord](https://discord.gg/JRNWnnva9g).

    
- Para contribuir a ALPS, por favor contacta a un miembro del [Consejo Directivo](#comité-directivo-comunitario-de-alps).

### Proceso de Toma de Decisiones 

Las contribuciones y cambios a ALPS ocurren mediante un modelo de consenso. Las propuestas de modificaciones serán revisadas tanto por los mantenedores como por los mantenedores principales una vez publicadas en el repositorio de GitHub. Las modificaciones se aceptarán si no se hacen comentarios dentro de 6 semanas o si todos los mantenedores están de acuerdo con las modificaciones. Las decisiones sobre propuestas controvertidas pueden apelarse ante el consejo directivo.

### Proceso de Contribución

El punto de partida práctico para cualquier contribución es la página [Contribuir a ALPS](../contribute), que describe el flujo de trabajo y lo que buscamos en un pull request. El archivo [CONTRIBUTING.md](https://github.com/ALPSim/ALPS/blob/master/CONTRIBUTING.md) en el repositorio de ALPS tiene todos los detalles técnicos completos (instrucciones de compilación, convenciones de ramas, estilo de código).

Las contribuciones en todos los niveles son bienvenidas — desde reportes de bugs y mejoras de tutoriales hasta nuevos métodos de simulación. Consulta la [página de incorporación](../onboard) para una descripción de los niveles de contribución y qué esperar en cada uno.

Para contribuciones de nuevas aplicaciones o bibliotecas de simulación, por favor contacta a un miembro del [Consejo Directivo](#comité-directivo-comunitario-de-alps) para discutir la incorporación. Los colaboradores se unen al equipo de ALPS a través de GitHub y acuerdan un compromiso de tiempo de mantenimiento modesto (correcciones de bugs, actualizaciones de dependencias, apoyo comunitario) para ayudar a sostener el proyecto. Todo el código se publica bajo la licencia de código abierto MIT.

La participación comunitaria tiene lugar a través de [talleres regulares de ALPS](../../events) y del [servidor de Discord](https://discord.gg/JRNWnnva9g).

Los lanzamientos de ALPS van acompañados de una publicación de anuncio. Los colaboradores activos se añaden como coautores; el Consejo Directivo decide la lista de autores.
